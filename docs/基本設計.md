# FOR FUTURE 25 — リニューアル基本設計書

> 作成日: 2026-07-10 / 対象: 特設サイト「FOR FUTURE 25」フルリニューアル
> 実装方式: **データ駆動（JSON + バニラJS / ビルドなし・GitHub Pages 継続）**

---

## 1. 目的・スコープ

### 1.1 背景と課題（現状）
- コンセプト（「九州から始まる、未来を作る25のストーリー」）は明確で演出も良いが、以下の構造的課題がある。
  - **25社ぶんの管理がベタ書き** … トップに実在6社＋ダミー約19枚（`YAMADA TARO`）。1社追加ごとに HTML 手編集が必要。
  - **詳細ページが1社=1ファイル** … `nishitetsu.html` 等が個別に存在。共通部（header/footer/head）が全ページに重複。
  - `alt` 空・OGP/description 無し・壊れた `head` 参照など、品質面の未整備。

### 1.2 このリニューアルのゴール
1. **25社を1つのデータ（JSON）で管理**し、一覧カードと詳細ページを自動生成する。
2. 詳細ページを **1テンプレートに統合**（1社=1ファイルを廃止）。
3. 見た目・アニメーションを刷新（トーンは踏襲：コーラル×白×黒カード）。
4. アクセシビリティ / SEO / OGP を整備。

### 1.3 制約・前提
- ホスティングは **GitHub Pages 継続**（`.github/workflows/static.yml`）。ビルド工程は追加しない。
- 依存は最小限（現状の jQuery は段階的に廃止し、バニラJSへ）。
- 既存の作り込んだレスポンシブ資産（`index.css` のグリッド等）は可能な限り活かす。

## 3. データ設計（中核）

### 3.1 配置
```
data/
  companies.json        ← 25社の配列。ここが唯一の情報源（Single Source of Truth）
```

### 3.2 スキーマ（1社 = 1オブジェクト）

```jsonc
{
  "id": "nishitetsu",              // URLスラッグ / 一意キー（英小文字・ハイフン）
  "order": 1,                       // 一覧の並び順
  "published": true,                // false なら一覧・詳細に出さない（Coming Soon制御）
  "category": "INDUSTRY",           // カード左の縦帯ラベル（§3.3 の定義値）
  "region": "長崎",                 // 九州テーマ用（長崎 / 佐賀 …）

  "person": {
    "name_ja": "丹山 裕和",
    "name_roman": "NIYAMA HIROKAZU",// カード＆詳細見出しのローマ字
    "title": "代表取締役社長"        // 役職
  },
  "company": {
    "name": "西鉄車体技術株式会社",
    "url": "https://www.nabt.co.jp/"
  },
  "tagline": "続けることで才能を越えられる",  // ヒーロー下の一言＝多くは最終セクション見出し

  "images": {
    "card":  "img/nishitetsu/top.jpg",             // 一覧カードの表
    "logo":  "img/nishitetsu/nishitetsulogo.png",  // カードのホバー裏（ロゴ）
    "hero":  "img/nishitetsu/bumon.jpg"            // 詳細トップの人物/イメージ
  },

  "sections": [                     // 詳細本文。見出し＋段落＋画像の繰り返し
    {
      "heading": "九州随一の技術力を誇る車体修理・改造会社",
      "body": [                     // 段落配列（空行区切りを配列で表現）
        "北部九州を拠点に大型バスや特殊車両の…"
      ],
      "image": "img/nishitetsu/company.jpg"
    }
    // … 通常4セクション程度
  ],

  "profile": {                      // 最終セクションのプロフィール枠
    "name": "丹山 裕和",
    "position": "西鉄車体技術株式会社　代表取締役社長",

    "description": "西鉄車体技術・丹山裕和の挑戦。九州随一の…",
    "ogImage": "img/nishitetsu/top.jpg"
  }
```

### 3.3 区分（category）の定義値
現状使用値: `FINANCIAL` / `INDUSTRY` / `VOLLEYBALL` / `WAGASHI`。

- 推奨: 表示ラベル（英字）と内部値を分けず、`category` は定義済みの列挙値のみ許可。

## 4. 画面設計

### 4.1 P-01 トップ（`index.html`）
構成（上から）:
1. **ヒーロー** … 画像モザイクグリッド（既存流用）＋順次フェードイン＋浮遊。
2. **ABOUT** … コンセプト文。スクロールで登場。
3. **STORY 一覧** … `#article-grid` を空コンテナにして、`js/render-index.js` が `published:true` の企業を `order` 順にカード生成。ダミーカードは全廃。
4. **フッター** … 既存流用。

カードの HTML はテンプレート（`<template>` タグ）を用意し、JSで複製して差し込む。

### 4.2 P-02 企業詳細（`company.html`）
- URL: `company.html?id=<slug>`。`js/render-company.js` が `id` を読み、`companies.json` から該当社を取得して描画。
- 現行 `nishitetsu.html` / `SAGA.html` は既に **同一構造**（`top-wrap__img` → `about-wrap__body__section` × N → `profile`）。この構造をそのままテンプレ化するので**見た目は現行維持で移行できる**。
- `id` が無い/不正/`published:false` の場合は 404（P-05）へ。
- `<head>` の `title` / `description` / OGP は JS で該当社の値に差し替え。

### 4.3 共通パーツ
- header / footer は全ページ共通。**JSで注入**（`js/partials.js`）するか、当面は各静的ページに直書きのままにするかを §7 で選択。

### 5.3 CSS 構成（リファクタ方針）
現状 `main.css` / `index.css` / `yamada.css` / `sub.css` … に分散。目標:
```

  tokens.css       ← 変数・タイポ・カラー（renewal.css から分離）
  base.css         ← body, a, 共通レイアウト
  components.css   ← card / nav / section / profile …
  index.css        ← トップ固有（既存グリッド流用）
  company.css      ← 詳細固有（旧 yamada.css を整理）
```
> 一気に統合せず、まず `renewal.css` を土台に段階移行する（§7）。

## 6. アニメーション / インタラクション設計

すべて `prefers-reduced-motion` を尊重（実装済み）。

| # | 対象 | 動き | 実装 |
|---|------|------|------|
| A1 | 一覧カード / ABOUT / セクション | スクロールで下から順に登場（スタッガー） | IntersectionObserver（`reveal.js` 実装済み） |
| A2 | 企業カード | ホバーで浮き・影・画像ズーム・裏ロゴ拡大 | CSS（実装済み） |
| A3 | ヒーロー | 順次フェードイン→浮遊、タイトル帯グラデ流動 | CSS + JS（実装済み） |
| A4 | ページ遷移 | 読込フェードイン / 離脱フェードアウト | `reveal.js`（実装済み・**全ページ展開が必要**） |
| A5 | ナビ | ドロワー blur・リンク下線 | CSS（実装済み） |

## 7. 技術構成 / レンダリング方式

### 7.1 描画フロー
```
[ index.html ]
   └─ js/render-index.js
        1. fetch('data/companies.json')
        2. published:true を order 順にソート
        3. <template id="card-tmpl"> を複製してカード生成 → #article-grid へ
        4. 生成後に reveal.js の監視対象へ登録

[ company.html?id=nishitetsu ]
   └─ js/render-company.js
        1. URLSearchParams で id 取得
        2. companies.json から該当社を検索（無ければ 404）
        3. hero / sections / profile を描画
        4. <head> の title/description/OGP を差し替え
```

### 7.2 SEO 上のトレードオフと対策
- クライアント描画のため、初期HTMLに本文が無い＝クローラ非対応の懸念。
- **対策（軽い順）**:
  1. `title`/`description`/OGP を JS で確実に注入（最低ライン）。
  2. 各詳細に静的な代替リンク（旧URL）とサイトマップ `sitemap.xml` を用意。
  3. （将来）必要なら Node の小さな **プリレンダースクリプト**で `company/<id>.html` を書き出し、GitHub Actions に組み込む（ビルドは任意・後付け可能な設計にしておく）。
- 主要SNS（OGP）と一覧トップのSEOは①で担保できるため、初期リリースは①で開始する。

### 7.3 依存
- jQuery を廃止し、`header.js` 等をバニラJSへ書き換え（`reveal.js` は既にバニラ）。
- 画像は `loading="lazy"` ＋ WebP 統一を推奨（別タスク）。

## 9. 非機能要件

| 項目 | 方針 |
|------|------|
| パフォーマンス | 画像 lazyload / WebP、JSON1回fetch、CSS/JS最小化 |
| アクセシビリティ | `alt` を人物・企業名に、`prefers-reduced-motion` 尊重、コントラスト確保、キーボード操作 |
| SEO | `description`/OGP/`sitemap.xml`、意味あるURL（`?id=`） |
| ブラウザ | 最新2世代の Chrome/Safari/Edge/Firefox、iOS/Android Safari |
| 保守性 | 企業追加は `companies.json` に1ブロック追記のみ |
